  Default (mode 1) runs a Pépin primality test or Fermat-PRP test, followed by a Suyama cofactor

  Fermat_exponent          Should be a whole number from 1 to 30
  factor(s)                Whole number(s) dividing a Fermat number F = 2^2^m + 1, m < 31

  Mersenne_exponent        Should be a whole number greater than 2

                           Up to 15 factors are supported (the most known is currently 12)

 -c  filename              Same as -cpr or --check-proof, but -c may be combined with other flags

 -upr  filename  or        Mode 3: Bypass the Pépin test, read the Suyama A residue from a mprime
 --use-proof  filename     proof file and use it to complete the Suyama test
 -u  filename              Same as -upr or --use-proof, but -u may be combined with other flags

 -a, --all-residues        Print residues for every squaring (not recommended for large numbers)
 -b, --binary              Print full residue in binary (also not recommended for large numbers)
 -d, --debug               Print debug information
 -h, --help                Print basic help and exit (-hv and -h -sv give increasingly more info)
 -i, --interim-residues    Special flag for printing certain interim residues (e.g. F8, F17, F22)
 -k, --known-factors       Use currently known prime factors of Fermat numbers (as of 2012).
                           If you have a Mersenne proof, the header may list known factors to use.
                       **  Do NOT use -k if you believe you have a new factor to test.   **

 -m, --mod-c               Special flag for reducing Suyama A and B values modulo the cofactor C
 -o, --octal               Additionally print Selfridge-Hurwitz residues in octal
 -v, --verbose             Print more verbose information
 -x, --hex, --hexadecimal  Also print Selfridge-Hurwitz residues in hexadecimal
 -j, --report-json         Enable JSON manual results reporting for Mersenne cofactor tests:
 -w, --user  string        This adds a username to the JSON report.
 -q, --computer  string    This adds the name of which computer ran the test to the JSON report.

 -p, --iterations  number  Print progress every multiple iterations, instead of default 10%
 -sep, --separator         Print a separator line at the end of a run to better see multiple runs
 -t, --threads  number     Specifies the number of threads to use in the gwnum library

----------------------------------------------------------------------------------------------------

  cofact was originally written by Gary B. Gostin to confirm cofactor tests of the Fermat numbers
  from F25 up to F30 (the largest of which reaches about 320 million digits in size).
  This version has been extensively revised, with various custom modifications by Catherine X. 
  Cowie for outputting data for her investigation of historical compositeness tests, published at:
      https://64ordle.au/fermat/historical/

  These modifications include:

  This version also supports Fermat-PRP testing of Mersenne numbers and cofactors.

  Flag -i to output interim residues for particular Fermat numbers:

      F17, 20th iteration (cf. Selfridge & Hurwitz, 1964);

      F22, 13 interim values from 126,000 to 4,182,000 squarings (cf. Trevisan & Carvalho, 1995);

      Morehead and Western (1905, 1909) used a method for evaluating the residue m+1 squarings 
      before the final residue. If the -i flag is set for any Fermat number greater than F2, this 
      interim residue for the (2^m - m - 2)th squaring will be printed.

  If the -p flag is also used with -i, then residues will also be printed at each set interval.

  Flag -m for reducing the Suyama A and B values by mod C, prior to evaluating (A - B) mod C
  (cf. Crandall, Doenias, et al., 1995);

  Options -o and -x for printing octal and hex representations of Selfridge-Hurwitz residues;

  Flag -b for printing the entire binary representation of the residue;

  Undocumented super-verbose flag -sv for outputting full residues of Pépin tests and from saved 
  proofs, full values of cofactors, outputting the binary representations of cofactors (yikes), 
  and other what-not.

  Use with care! The super-verbose flag -sv also includes the normal -v level of verbosity, and 
  switches on -d (debug), -i, -m, -o, and -x flags as well. It is not a good idea to run -sv on 
  Fermat numbers greater than F10 unless you need or want full listings of cofact's doings.

  Any of the following flags may be run together as one argument, i.e. as -abcdhijkmopqtuvwxz:
    -a, -b, -i, -m, -o, -p, -x  for various modifications to the Pépin or Suyama test output;
    -c or -u  filename          to either check or use a proof file (but you cannot specify both);
    -d, -v                      for debug or verbose modes;
    -h or -hv                   for help or verbose help;
    -k                          to use the known factors for a Fermat number, or if you have a
                                proof of a Mersenne number, the file header may list known primes;
    -j                          producing a JSON report of a cofactor test (Mersennes only);
    -w or -q  string            adding user or computer strings to JSON report;
    -t  threads                 to use multithreading for the Pépin test;
    -z  base                    to use a non-standard base for the Pépin or Suyama test.
  Flags that require additional parameters as arguments (-c, -p, -q, -t, -u, -w, -z) cannot be
  combined as a single flag. In addition, if both -c and -u are separately specified (to check
  or utilise a proof) cofact will report the error and bypass the Pépin or Fermat-PRP test.
  Not all possible bases are supported for the Pépin test; cofact will attempt to revert to a
  Fermat-PRP test if the chosen base is unsuitable.
  If -a with either -b and/or -i are used together, cofact will tacitly ignore -b and/or -i.

  Conversely, the -cpr, -sep, -sv, and -upr flags must be entered separately.

  The following numbers of known factors as of 2012 are programmed with the -k flag:
      F12: 6 known factors
      F13: 4 known factors
      F15, F19, F25: 3 known factors
      F16, F17, F18, F27, F30, [F36]: 2 known factors
      F14, F21, F22, F23, F26, F28, F29, [F31, F32]: 1 known factor
      F5, F6, F7, F8, F9, F10, F11: all factors

****  I cannot stress enough - if you are testing for a newly discovered factor,  ****
****  do NOT use the -k flag!                                                     ****

  Gary Gostin's version of cofact will not allow composite factors to be tested. This version does
  allow composite factors, and will show whether any known prime factors divide them. (Frequently
  ECM is able to find composite factors, which turn out to be products of known factors.)

  If you are running cofactor tests on Mersennes, cofact will have no known factors stored. If
  you have a proof file, the header may list known factors which cofact can read, using the -k
  flag, but this cannot certainly account for all possibilities or discovery of new factors.
  cofact will inform you whether a factor you supply is composite, in which case you are strongly
  encouraged to check its divisibility by any known factors of that Mersenne.

  Other notes:

      You cannot test F0. (It doesn't need testing. Try cofact 0 if you don't believe me.)
      You cannot test F31 or greater with cofact. (Yet.)
      Testing F30 in mode 1 or 2 requires an x86 family processor with AVX-512 extensions.
      Using proof files for F21 and above is highly advisable. Proofs for F12 to F29 (which
      are typically named Fxx.proof, substituting the exponent) are available from:
          https://64ordle.au/fermat/             (e.g. https://64ordle.au/fermat/F12.proof)

  Results of historical tests cited in my essay:

  cofact -a 5 641            1877, demonstration of Théophile Pépin's theorem with F5;
  cofact -ov 7               1905, Morehead and Western separately test F7;
  cofact -iov 8              1909, Morehead and Western together test F8;
  cofact -v 10               1952, Robinson tests F10 with the SWAC;
  cofact -o 13               1960, Paxson tests F13 with an IBM 7090 mainframe;
  cofact -o 14               1961, Hurwitz and Selfridge test F14 with a (possibly overclocked?)
  cofact -io 17                    IBM 7090, and provide an interim residue for F17;
  cofact -ot 4 20            1988, Buell and Young test F20 with a couple of Cray supercomputers;
  cofact -it 4 22            1993, Trevisan and Carvalho obtain interim residues of F22 on a Cray,
  cofact -mku F21.proof 21         and Crandall, Doenias, et al., run Suyama tests on F19 and F21;
  cofact -m 19 70525124609 646730219521      (these two Suyama tests work faster using VDF proofs)
  cofact -t 4 24             1999, Crandall, Mayer, and Papadopoulos find F24 composite;
  cofact -ku F28.proof 28    2022, Mayer announced results of Suyama tests of F28, F29, and F30.
